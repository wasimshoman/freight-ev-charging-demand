# Trip-Based EV Charging Demand Model for Long-Haul Freight

A Python pipeline that estimates the spatial demand for electric vehicle (EV) charging infrastructure along European long-haul freight corridors. The model traces individual truck trips across the road network, applies mandatory EU rest and break regulations to identify where stops occur, and computes the charging demand at each stop location weighted by freight flow volumes.

## Methodology

The pipeline implements a trip-based bottom-up approach to quantify where and how much EV charging infrastructure is needed to support the electrification of long-haul road freight in Europe. The full methodological details are described in:

> Shoman, W., Yeh, S., & Kaberger, T. (2023). Estimation of battery electric truck  
> charging demand and infrastructure for long-haul trips in Europe.  
> *Transportation Research Part D: Transport and Environment*, 121, 103832.  
> https://www.sciencedirect.com/science/article/pii/S1361920923002225

### Summary

**1. Route generation** (`01_generate_route_points.py`)

Origin-destination (OD) pairs from the ETISPlus freight flow model are filtered to long-haul routes (straight-line distance > 200 km, annual flow > 50 tonnes). For each qualifying OD pair the road path is reconstructed from a pre-routed edge sequence in the network data, and GPS points are placed every 10 km along the route using geodesic (WGS84 ellipsoid) interpolation. Each point inherits road attributes from the nearest network edge (speed class, traffic counts, edge ID).

**2. Rest and break rule application** (`02_apply_rest_break_rules.py`)

EU Regulation 561/2006 mandatory rest and break rules are applied to each route by accumulating driving time along the point sequence:

- *Single-driver trips* (4.5–15 h estimated duration): one 45-minute break followed by a 9-hour daily rest after the next 4.5-hour block.
- *Two-driver trips* (≥ 15 h estimated duration): up to three 45-minute breaks followed by a 9-hour daily rest.

Ferry crossings and road segments without charging infrastructure do not accumulate driving time. The location, energy consumed since the previous stop, and OD flow data are recorded for every break and rest event across all routes.

**3. Spatial aggregation to a 25×25 km grid** (`03_aggregate_to_grid.py`)

Stop and rest events are spatially joined to a regular 25×25 km grid (EPSG:3035). The number of fast chargers (CCS, 30-minute and 60-minute sessions) and slow chargers (MCS, overnight) required per grid cell is determined using a pre-computed M/G/c queuing model, calibrated to achieve an acceptable queue length at peak demand. Results are also aggregated by country.

**4. Demand by origin zone** (`04_summarise_by_origin.py`)

Stop events are grouped by the NUTS-3 origin zone of the trip to quantify how much charging demand is generated by freight departing from each region.

**5. Demand by country** (`05_summarise_by_country.py`)

Daily truck demand is re-derived from raw annual tonnage flows and aggregated by origin country, providing a country-level breakdown of EV charging load.

### Input data

The pipeline requires two sets of input data:

**Freight model data** — available for download from Zenodo:

> Shoman, W. (2023). *Data for: Estimation of battery electric truck charging demand  
> and infrastructure for long-haul trips in Europe* [Dataset].  
> Zenodo. https://zenodo.org/records/7867172

This dataset includes the OD flow matrix, road network nodes and edges, NUTS-3 zone centroids, queuing model tables, and country boundary files required to run the pipeline.

**Road network data** — the ETISPlus network is distributed separately. The files expected by this pipeline are:

| File | Description |
|---|---|
| `01_Trucktrafficflow.csv` | OD flow matrix — annual truck and tonnage flows between NUTS-3 zones |
| `02_NUTS-3-Regions.csv` | NUTS-3 zone centroids (ETISPlus zone IDs and coordinates) |
| `03_network-nodes.csv` | Road network node coordinates |
| `Updated_04_network-edges.csv` | Road network edges with speed class, traffic counts, and routed edge paths |
| `Centroids.shp` | NUTS-3 centroid shapefile (provides CRS and geometry) |
| `queue_30min.csv` | M/G/c queuing model table for 30-minute fast charger sessions |
| `queue_60min.csv` | M/G/c queuing model table for 60-minute fast charger sessions |
| `EU_contriesWithTurk.shp` | EU + Turkey country boundary polygons |

## Configuration

All file paths and model parameters are defined in `config.py`. Edit this file before running any pipeline script:

```python
# Input data paths — set these to match your local data location
FLOW_CSV         = r'path/to/01_Trucktrafficflow.csv'
NUTS3_CSV        = r'path/to/02_NUTS-3-Regions.csv'
NODES_CSV        = r'path/to/03_network-nodes.csv'
EDGES_CSV        = r'path/to/Updated_04_network-edges.csv'
CENTROIDS_SHP    = r'path/to/Centroids.shp'
QUEUE_30MIN_CSV  = r'path/to/queue_30min.csv'
QUEUE_60MIN_CSV  = r'path/to/queue_60min.csv'
EU_COUNTRIES_SHP = r'path/to/EU_contriesWithTurk.shp'
```

Output files are written to the `outputs/` sub-folder within this project directory. This path is configured automatically and does not need to be changed.

Key model parameters (can be adjusted in `config.py`):

| Parameter | Default | Description |
|---|---|---|
| `POINT_SPACING_KM` | 10 | Distance between generated route points (km) |
| `MIN_HAUL_KM` | 200 | Minimum OD distance for long-haul classification (km) |
| `EV_SHARE_MAIN` | 0.15 | Main scenario fleet electrification share |
| `GRID_SIZE_M` | 25,000 | Spatial aggregation grid cell size (m) |
| `N_WORKERS` | None | Parallel workers for scripts 01 and 02 (None = all cores) |

## Running the pipeline

Run scripts from inside the `trip_based_model/` directory in order:

```bash
# Step 1 — generate route point shapefiles (~hours, parallelised)
python 01_generate_route_points.py

# Step 2 — apply rest/break rules and compute charging demand (~hours, parallelised)
python 02_apply_rest_break_rules.py

# Steps 3, 4, and 5 are independent and can run in parallel after step 2

# Step 3 — aggregate to 25×25 km grid and size chargers (~minutes)
python 03_aggregate_to_grid.py

# Step 4 — summarise demand by origin zone (~seconds)
python 04_summarise_by_origin.py

# Step 5 — summarise demand by country (~seconds)
python 05_summarise_by_country.py
```

Scripts 01 and 02 use Python's `multiprocessing` module and must be run as `__main__` (i.e., directly via `python script.py`, not imported). Script 01 is crash-safe: if interrupted it will skip already-completed routes on restart.
